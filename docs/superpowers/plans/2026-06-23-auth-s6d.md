## Phase 1 ‚Äî close the loop (platform-agnostic; enter-code path)

1. **API `GET /device/pending`** ‚Äî `apps/api/src/auth/device.ts` (lookup helper)
   + route in `apps/api/src/app.ts`. Session-auth, shared `account:approve:<sub>`
   rate-limit + lockout, uniform 404, audit `device.lookup`. Returns `origin_kind`
   (`datacenter|residential|unknown`) via a **bundled cloud/datacenter CIDR
   classifier** (`apps/api/src/auth/origin.ts`, no vendor) ‚Äî ADR 0011 ¬ß7 intent.
   Regenerate the Vercel bundle (`apps/api/api/index.js`). **Tests:** `apps/api/
   test/device-pending.test.ts` ‚Äî hit/miss/expired/lockout/401/audit + CIDR
   classification (datacenter vs residential vs unknown).
1b. **Central scope gate** ‚Äî `requireScope(resource, action)` helper; read routes
   (`GET /‚Ä¶/cards`, `/‚Ä¶/sync`) enforce `content:read` (closes the read-enforcement
   gap). ADR 0029 enforcement half; grant *selection* deferred to the content
   slice. **Tests:** read-without-`content:read` ‚Üí 403; write still gated.
2. **AuthClient** ‚Äî `devicePending` / `deviceApprove` / `deviceDeny` + `PendingDevice`
   in `AuthClient.kt`; MockEngine tests (`AuthClientTest.kt`).
3. **State/actions/reducer** ‚Äî `Route.EnterCode|AuthorizeDevice`, `pendingDevice`/
   `deviceBusy`/`deviceError`, the device actions, reducer arms. `ReducerTest`
   cases incl. expired/denied/failed.
4. **AuthEngine** ‚Äî `lookupDevice`/`approveDevice`/`denyDevice` (mutex +
   `callWithRefresh`); `AuthEngineTest`.
5. **Screens** ‚Äî `DeviceApprovalScreens.kt`: `EnterCodeScreen`,
   `AuthorizeDeviceScreen` (owner-family selector via `whoami.families`
   role=owner; static row when 1; **datacenter-origin warning banner** when
   `origin_kind=="datacenter"`; scope row informational per ADR 0029),
   `DeviceDeniedScreen`, `DeviceExpiredScreen`. Snapshot tests vs the A8b mockup
   PNGs (light+dark) incl. the datacenter-warning variant.
6. **FeedApp + entry** ‚Äî two `when(route)` arms; CTA wired from `FamilyNullState`
   + a "Connect a device" affordance in `DevicesScreen`.
7. **CLI QR** ‚Äî `zxing-core` in `apps/cli/build.gradle.kts`; render
   `verification_uri_complete` in `deviceLogin` above the text code; keep text.
   `./gradlew build`.
7b. **CLI refresh-token ‚Üí OS keychain** ‚Äî move the 45-day refresh token out of
   plaintext `~/.config/dayfold/credentials.json` into the OS credential store
   (`expect/actual`; macOS Keychain / libsecret / Windows Credential Manager;
   headless fallback `0600` behind `--allow-env-key` + warning per 07-cli). Access
   token + non-secret config may stay in the config file. **Tests:** save/load
   round-trip; absent-keychain fallback path.
8. **E2E round-trip (desktop)** ‚Äî dev-auth API: `dayfold login` ‚Üí enter code ‚Üí
   approve ‚Üí "logged in" ‚Üí `dayfold push` 200. Verify via `[redux]` log.

**Phase-1 gate:** loop closes via typed code on desktop + Android; suites green.

## Escalations / dependencies
- **ADR 0029 (per-hub scope grants)** ‚Äî Proposed; operator acceptance gates the
  scope-*selection* build (which rides with the content slice below). S6-D ships the
  interim global `content:read+write` grant; the scope row is informational.
- **Precise geo/ASN** ‚Äî still deferred (vendor + cost). The no-vendor datacenter
  CIDR heuristic (step 1) covers the ADR 0011 ¬ß7 intent for now.
- **Scan/deep-link mockups + sign-off** (`designs/DESIGN-BRIEF-device-scan.md`) ‚Äî
  gates step 10. Authoring via the `claude_design` MCP connector requires the
  connector be authorized (`/design-login` ‚Üí `user:design:read/write`).
- **`zxing-core`** (CLI) + **CameraX/ML Kit** (Android) + **AVFoundation** (iOS) +
  **OS keychain libs** (libsecret/Keychain/WinCred for step 7b) dependencies ‚Äî
  minor maintenance-burden adds; ML Kit barcode is on-device (no network, no cost).

## Next slice (planned after S6-D closes the login loop)
**Content-API + CLI content verbs + per-hub scoping** (operator-chosen 2026-06-23):
- API: hub/section/block endpoints (read + write), each behind `requireScope`.
- CLI: `pull` / `hub get|archive|rm` / `status` / `push --dry-run|--diff` (07-cli),
  i.e. the CLI becomes a real content read+write client (today it only `PUT`s one
  card). `whoami` shows family + scope + label.
- Approval: `credential_grants` table + per-hub read/write picker on
  `AuthorizeDevice` (ADR 0029 item 3) ‚Äî replaces the informational scope row.
This satisfies the operator requirement "hubs and user content read+written from
CLI" + "user can scope a grant per resource"; it is **out of S6-D scope** and gets
its own spec ‚Üí plan ‚Üí build cycle.

## Risks
- Deep-link verification (`assetlinks.json`/AASA) is fiddly to verify in CI ‚Äî
  manual device check + keep enter-code as the guaranteed fallback.
- iOS camera via `UIKitView` interop is the highest-effort actual ‚Äî Android +
  enter-code can ship first if iOS lags (desktop already degrades to enter-code).
- Cold-install resume (deep-link before sign-in) is the subtle path ‚Äî explicit
  reducer test for stash‚Üíonboard‚Üíresume.
